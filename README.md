# showcase

GitHub Pages로 배포하기 위한 개인 showcase 정적 페이지입니다.

## 구조

- `index.html`: 프로젝트 소개, resume, notes, contact 섹션
- `styles.css`: monospace 기반 Terminal UI 스타일
- `script.js`: 히어로 영역의 간단한 command 텍스트 전환
- `.github/workflows/static.yml`: `main` 브랜치 push 시 GitHub Pages 배포

## 수정 포인트

1. `index.html`의 `Surajang`, 역할, 프로젝트 카드, 이력서 내용을 실제 정보로 교체합니다.
2. 프로젝트별 `details` 링크를 GitHub 저장소, 블로그 글, 데모 URL로 연결합니다.
3. `mailto:hello@example.com`과 GitHub, LinkedIn 링크를 실제 주소로 바꿉니다.
4. GitHub 저장소 설정에서 Pages source를 `GitHub Actions`로 지정합니다.
